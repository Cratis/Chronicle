# CHR0015: Reactor must not inject IEventLog

## Rule Description

A class implementing `IReactor` must not inject `IEventLog` via its constructor. Reactors observe events and should react by issuing commands through `ICommandPipeline`, not by appending events directly.

## Severity

Error

## Example

### Violation

```csharp
using Cratis.Chronicle.EventSequences;
using Cratis.Chronicle.Reactors;

public class UserSyncReactor : IReactor
{
    readonly IEventLog _eventLog;

    public UserSyncReactor(IEventLog eventLog)
    {
        _eventLog = eventLog;
    }
}
```

### Fix

```csharp
using Cratis.Chronicle.Reactors;

public class UserSyncReactor : IReactor
{
    readonly ICommandPipeline _commandPipeline;

    public UserSyncReactor(ICommandPipeline commandPipeline)
    {
        _commandPipeline = commandPipeline;
    }
}
```

## Why This Rule Exists

Reactors are observers that react to domain events. The appropriate way for a reactor to cause state changes is to issue commands through `ICommandPipeline`. Injecting `IEventLog` creates a direct coupling that bypasses the command pipeline, violates the observer/command separation of concerns, and can lead to causality tracking issues.
