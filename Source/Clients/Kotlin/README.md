# Kotlin Chronicle Contracts

This package provides generated gRPC contracts for Chronicle and a Kotlin-friendly connection layer.

## Why Java Files Are Generated

It is expected that generation produces both Java and Kotlin files.

- Java files under `build/generated/source/proto/main/grpc` are gRPC infrastructure classes generated by `protoc-gen-grpc-java`.
- Kotlin coroutine stubs under `build/generated/source/proto/main/grpckt` are generated by `protoc-gen-grpc-kotlin` and are the primary client API for Kotlin callers.
- Kotlin message DSL files under `build/generated/source/proto/main/kotlin` provide Kotlin-native builders and helpers for proto messages.

The Kotlin coroutine stubs depend on gRPC descriptors and support code produced by the Java generator. This is the standard gRPC Kotlin generation model.

## What To Use In Kotlin Code

Prefer the coroutine stubs from the `grpckt` output and the connection abstractions in:

- `src/main/kotlin/io/cratis/chronicle/contracts/ChronicleConnection.kt`
- `src/main/kotlin/io/cratis/chronicle/contracts/ChronicleConnectionString.kt`

Avoid using generated Java stubs directly unless you specifically need Java-style APIs.

## Local Generation Workflow

1. Generate committed proto files from Contracts:
   - `Source/Kernel/Protobuf/generate-protos.sh`
2. Generate Kotlin client artifacts from local proto files:
   - `Source/Clients/Kotlin/generate-protos.sh`

The Kotlin script syncs `Source/Kernel/Protobuf` into `src/main/proto` and runs `generateProto`.

## Notes

- Generated output lives under `build/generated` and is not intended to be edited manually.
- `src/main/proto` is generated input for Kotlin generation and is ignored in git.
